# C-- 语言编译器前端 (Compiler Frontend for C--)

## 1. 项目简介

本项目为 2026 春季编译原理课程大作业，实现了一个完整的 C-- 语言（C 语言子集）编译器前端。编译器能够完成从源代码输入，到词法分析、语法分析（基于 SLR(1) 算法），再到符合 LLVM IR 规范的中间代码生成的完整编译流程。

项目已完成所有必做任务，覆盖 C-- 语言核心语法特性，包括：

- 变量声明与初始化
- 常量定义
- 函数定义与调用
- 算术运算与逻辑运算
- 条件分支控制
- 作用域与变量遮蔽
- LLVM IR 中间代码生成

此外，项目还提供了自动化测试脚本，可一键运行全部测试用例并生成测试报告。

---

# 2. 工程结构

工程采用模块化设计，每个编译阶段对应独立模块，结构清晰、易于扩展。

| 文件名 | 说明 |
| :--- | :--- |
| **lexer.h / lexer.cpp** | **词法分析器**：基于 DFA（确定有限自动机）识别 Token，输出二元属性并生成符号表。 |
| **parser.h / parser.cpp** | **语法分析器**：实现 **SLR(1)** 语法分析算法，构建 AST 并输出规约过程。 |
| **ir_visitor.h / ir_visitor.cpp** | **LLVM IR 中间代码生成模块**：采用 Visitor 模式遍历 AST 并生成 LLVM IR。 |
| **main.cpp** | **主程序入口**：控制整个编译流程，并关闭缓冲区保证 PowerShell 实时输出。 |
| **run_tests.ps1** | **自动化测试脚本**：批量运行测试用例、自动比对结果并生成测试报告。 |
| **Readme.md** | 项目说明文档。 |
| **test_results/** | 自动测试输出目录（运行测试脚本后自动生成）。 |

# 3. 核心功能实现

## 3.1 词法分析（Phase 1）

### 实现机制

词法分析器基于 DFA（确定有限自动机）实现，对输入字符流逐字符扫描并识别 Token。

### 支持的 Token 类型

#### 关键字（KW，不区分大小写）

```text
int
void
return
const
main
float
if
else
```

#### 标识符（IDN）

符合 C 语言规范：

```text
[a-zA-Z_][a-zA-Z_0-9]*
```

#### 运算符（OP）

```text
+
-
*
/
%
=
>
<
==
<=
>=
!=
&&
||
!
```

#### 界符（SE）

```text
(
)
{
}
;
,
```

#### 常量类型

- 整型常量（INT）
- 浮点型常量（FLOAT）

### 高级特性

支持：

- 单行注释 `//`
- 多行注释 `/* */`
- 任意空白字符自动忽略

### 输出格式

输出单词符号二元属性：

```text
<种别名, 种别码>
```

例如：

```text
int <KW,1>
a <IDN,a>
= <OP,11>
10 <INT,10>
```

---

## 3.2 语法分析（Phase 2 - SLR）

### 算法方案

采用自下而上的 **SLR(1)** 语法分析算法。

### 核心实现内容

项目完整实现：

- LR(0) 项目集规范族构造
- FIRST 集计算
- FOLLOW 集计算
- SLR 分析表生成
- ACTION / GOTO 表驱动分析

### 状态管理

分析过程中维护：

- 状态栈
- 符号栈

根据分析表执行：

- Shift（移进）
- Reduction（规约）
- Accept（接受）
- Error（错误）

### 关键优化

实现了 LL(2) 前看机制，用于解决：

```text
int a;
int func(...)
```

中 `int` 既可能是变量类型也可能是函数返回类型导致的 SLR(1) 冲突问题。

### 支持的语法结构

支持：

- 全局变量声明
- 局部变量声明
- 常量定义
- 函数定义与调用
- 单目 / 双目运算
- if / else 分支控制
- 多层作用域嵌套

---

## 3.3 LLVM IR 中间代码生成（Phase 3）

### 设计模式

采用 Visitor（访问者）模式遍历 AST。

### 输出格式

生成标准 LLVM IR（`.ll`）中间代码。

### 支持的 IR 指令

支持生成：

- 全局变量定义
- 局部变量分配（alloca）
- store / load
- 算术运算
- 逻辑运算
- 条件跳转
- 函数定义
- 函数调用
- return 指令

### 高级特性

实现：

- 逻辑运算短路求值
- 多层作用域符号表管理
- 自动生成临时变量
- 自动生成标签（Label）

# 4. 快速开始

## 4.1 环境要求

- Windows 10 / 11
- GCC 8.1.0（MinGW-W64）或更高版本
- 支持 C++17
- PowerShell 5.1 或更高版本

---

## 4.2 手动编译与运行

在 PowerShell 中执行：

```powershell
# 删除旧程序并重新编译（静态链接）

g++ -std=c++17 -Wall -g -static lexer.cpp parser.cpp ir_visitor.cpp main.cpp -o mycompiler.exe

# 运行编译器

.\mycompiler.exe
```

---

## 4.3 调整输入源代码

若需测试不同代码，可修改 `main.cpp` 中：

```cpp
std::string code = "int main() { return 0; }";
```

修改字符串内容即可测试不同 C-- 源代码。

---

## 4.4 自动化测试（推荐）

项目提供完整自动测试脚本：

```powershell
.\run_tests.ps1
```

可自动运行全部测试用例并生成详细报告。

### 4.4.1 自动测试脚本功能

脚本支持：

- 自动备份原始 `main.cpp`
- 自动编译程序
- 批量运行测试用例
- 自动比对输出结果
- 生成详细日志
- 输出测试汇总报告

### 4.4.2 运行方式

```powershell
.\run_tests.ps1
```

### 4.4.3 测试结果说明

运行后将自动生成：

```text
test_results/
```

目录。

每个测试用例包含：

```text
[用例名]_actual.txt
[用例名]_expected.txt
[用例名]_compile_error.txt（如有）
```

全部通过时控制台会输出：

```text
🎉 所有测试用例全部通过！
```
