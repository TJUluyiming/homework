\# C-- 语言编译器前端 (Compiler Frontend for C--)



\## 1. 项目简介

本项目为 2026 春季编译原理课程大作业。实现了一个针对 C-- 语言（C 语言子集）的编译器前端，能够完成从源代码到词法分析、语法分析（基于 SLR 算法），并最终生成符合 LLVM IR 规范的中间代码。



\---



\## 2. 工程结构

工程遵循模块化设计，每个阶段对应独立的源代码文件，结构清晰：



| 文件名 | 说明 |

| :--- | :--- |

| \*\*lexer.h / .cpp\*\* | \*\*词法分析器\*\*：利用自动机理论识别 Token，输出二元属性。 |

| \*\*parser.h / .cpp\*\* | \*\*语法分析器\*\*：实现 \*\*SLR(1) 算法\*\*，构建抽象语法树 (AST)。 |

| \*\*ir\_visitor.h / .cpp\*\* | \*\*中间代码生成\*\*：采用 \*\*Visitor 模式\*\* 遍历 AST 并生成 LLVM IR。 |

| \*\*main.cpp\*\* | \*\*主入口\*\*：控制编译流程，禁用缓冲区以保证 PowerShell 实时输出。 |

| \*\*mycompiler.exe\*\* | 预编译的可执行程序。 |



\---



\## 3. 核心功能实现



\### 3.1 词法分析 (Phase 1)

\- \*\*实现机制\*\*：基于有限自动机 (DFA) 进行字符流扫描。

\- \*\*支持类型\*\*：关键字 (KW)、标识符 (IDN)、操作符 (OP)、分界符 (SE)、整型常量 (INT)。

\- \*\*输出格式\*\*：单词文本与二元属性 `<种别名, 种别码>`。



\### 3.2 语法分析 (Phase 2 - SLR)

\- \*\*算法方案\*\*：自下而上的 SLR(1) 语法分析。

\- \*\*状态管理\*\*：维护状态栈和符号栈，根据 ACTION/GOTO 表执行移进 (Shift) 和规约 (Reduction)。

\- \*\*输出结果\*\*：实时打印规约步骤，并最终构建出一棵完整的抽象语法树 (AST)。



\### 3.3 中间代码生成 (Phase 3)

\- \*\*设计模式\*\*：Visitor (访问者) 模式。

\- \*\*语义处理\*\*：在遍历 AST 过程中，识别 `varDef` 节点，提取变量名与初值。

\- \*\*IR 格式\*\*：生成符合 LLVM 标准的全局变量定义语句，例如：`@a = global i32 10`。
